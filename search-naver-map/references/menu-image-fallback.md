# 메뉴 이미지 fallback

메뉴 관련 질문은 `detail`의 구조화된 `data.menus`를 먼저 사용합니다. 이 문서의 절차는 구조화 메뉴만으로 질문에 답할 수 없을 때 에이전트가 수행하는 fallback입니다. CLI에 이미지 인식 기능을 추가하는 절차가 아닙니다.

## 1. 발동 여부 판단

다음 중 하나이면 fallback을 사용합니다.

- `data.menus`가 없거나, 비어 있거나, 의미 있는 메뉴명이 없음
- 사용자가 찾는 메뉴가 구조화 메뉴의 이름·설명에서 확인되지 않음
- 사용자가 가격을 물었지만 관련 메뉴의 가격이 없음
- 사용자가 요청한 메뉴 범위를 답하기에 구조화 결과가 명백히 부족함

메뉴 수가 적다는 이유만으로 자동 발동하지 않습니다. `partial`도 영업시간이나 피드 실패 때문일 수 있으므로 `errors[].operation`과 실제 메뉴 필드를 함께 봅니다. 구조화 메뉴가 질문을 충분히 답하면 이미지 조회를 생략합니다.

## 2. 공개 이미지 후보 수집

같은 Place ID로 상세 결과를 `full` 범위로 다시 받습니다. 메뉴 확인만 필요하면 불필요한 보조 조회를 줄일 수 있습니다.

```bash
bin/naver-place detail \
  --place "$PLACE_ID" \
  --view full \
  --no-feed \
  --no-hours \
  --output "$DETAIL_JSON"
```

`status`, `completeness`, `warnings`, `errors`를 다시 확인한 뒤 `data.photos`에서 후보를 고릅니다. 우선순위는 다음과 같습니다.

먼저 `full` 결과의 `data.menus`를 다시 확인합니다. `standard`에서 빠졌던 설명으로 질문에 답할 수 있게 됐다면 다운로드 없이 중단합니다. 사용할 수 있는 `data.photos` 후보가 없으면 이미지 fallback을 실행하지 못했다고 알리고 메뉴를 추정하지 않습니다.

1. `menu_board_images`: 메뉴판 사진
2. `menu_item_images`: 구조화 메뉴에 연결된 사진
3. `place_images`: 제목·설명·구도상 메뉴판일 가능성이 높은 매장 사진
4. `feed_images`: 이미 조회한 결과에 있고 `feed_title`이 메뉴·가격과 직접 관련된 사진

`blog_thumbnails`나 방문자 리뷰 사진은 사용하지 않습니다. 중복 URL을 제거하고 기본 8장까지만 검사합니다. 사용자가 빠짐없는 조사를 명시하지 않았다면 이 범위를 임의로 넓히지 않습니다. `origin`이 유효한 HTTPS 원본 URL이면 `url`보다 우선합니다.

## 3. 이미지 다운로드

선택한 공개 HTTPS URL만 임시 디렉터리에 내려받습니다. 인증 헤더, 쿠키, 로그인 세션, `.netrc`, 환경 프록시를 사용하지 않습니다. 이미지마다 출처 필드와 원본 URL을 함께 기록합니다.

```bash
curl --disable --noproxy '*' \
  --fail --silent --show-error --location \
  --proto '=https' --proto-redir '=https' --max-redirs 3 \
  --connect-timeout 10 --max-time 30 --max-filesize 15728640 \
  --output "$IMAGE_PATH" "$IMAGE_URL"
```

다운로드한 파일의 MIME 형식이 `image/*`인지 확인합니다. HTML, 빈 파일, 15 MiB 초과 파일, 깨진 파일은 비전 입력에서 제외합니다. 공개 요청이 거부되면 브라우저, 프록시, 로그인으로 우회하지 않습니다.

## 4. 비전으로 읽기

런타임의 이미지 보기·비전 기능으로 가능하면 원본 해상도를 확인합니다. 각 이미지에서 다음 항목만 읽습니다.

- 선명하게 보이는 메뉴명과 옵션·구성
- 해당 메뉴와 같은 행이나 영역에 명확히 연결된 가격
- 품절, 계절 한정, 인원 조건처럼 글자로 표시된 제한
- 판독하지 못한 영역과 잘린 글자

작은 글자는 확대해 한 번 더 확인하되 불명확한 글자를 보완해서 쓰지 않습니다. 음식 모양, 그릇, 사진 순서만으로 메뉴명·재료·판매 여부를 확정하지 않습니다. 메뉴판과 음식 사진이 충돌하면 하나로 합치지 말고 충돌을 알립니다.

비전 기능이 없거나 모든 후보가 판독 불가하면 fallback이 실패했다고 명시하고, 메뉴를 추정하지 않습니다.

## 5. 답변에 근거 표시

구조화 결과와 이미지 판독 결과를 구분합니다.

- `구조화 메뉴`: `data.menus`에서 직접 확인
- `메뉴판 이미지 판독`: 글자가 선명하게 보이는 공개 이미지에서 확인
- `사진상 추정`: 메뉴 확정 근거로 사용하지 않음

이미지로 확인한 항목에는 Place 링크와 이미지 원본 URL을 함께 제시합니다. 이미지 촬영·게시 시각을 알 수 없으면 현재 판매 여부나 최신 가격으로 단정하지 않습니다. 필요한 항목을 끝내 확인하지 못했다면 “공개된 구조화 메뉴와 검사한 이미지에서는 확인되지 않음”이라고 범위를 정확히 씁니다.
